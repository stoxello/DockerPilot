Manage Docker like a pilot, not a passenger

DockPilot is a C# and Blazor console for Docker and Docker Compose — the approachable stack workflow of Dockge, a built-in app catalog for one-click self-hosting, and progressively deeper environment management inspired by Portainer. One container, plain Compose files on disk, no vendor lock-in.

How it works

No SDK, no hidden state

DockPilot is an ASP.NET Core / Blazor (Interactive Server) app. Instead of a Docker SDK, it shells out to the docker and docker compose CLIs and parses their JSON output — the same thing you'd run by hand. Every stack it creates is written as an ordinary compose.yaml (and optional .env) under the stacks directory, so it stays fully inspectable and portable. Managed stacks are sandboxed by name to that directory, and only stacks DockPilot created can be edited or rewritten from the UI.

What it does

Features

DockPilot covers the full loop from discovering what's running to deploying, editing, backing up, and replicating it — on your local Docker host or across a fleet of remote agents.

Workspace
✦

App catalog

A searchable, built-in catalog of popular self-hosted apps with parameterized Compose previews and one-click deploy into an isolated stack directory.

▤

Stacks

Discovers every Compose project on the host, including DockPilot-managed stacks whose deploy never completed. Create, edit, and redeploy stacks from an in-app editor.

List with in-use detection; create and remove volumes.

≋

Events & audit log

A live Docker engine event feed alongside a DockPilot audit log of deploys, removals, edits, and sign-ins.

Resilience
B

Backup

Scheduled full and incremental backups of a stack's volumes and Compose configuration to the local host, with configurable retention and an option to stop the stack for a consistent snapshot.

R

Replication

Copy a stack to a registered DockPilot agent and keep a cold replica in sync on a schedule, ready for failover, with optional bandwidth capping.

Administration
●

Environments / Agents

Register remote DockPilot Agent instances and switch the Control Center between Docker hosts from a single top-bar selector.

⇪

Update checks

DockPilot compares its running image digest against the published one and flags the sidebar when a newer version is available.

⚿

Registry credentials

Sign Docker in to a private registry (GHCR, Docker Hub, or custom) on the local host or a specific agent, so private images pull cleanly during installs and updates.

Install

Quick start with Docker

This is the recommended way to run DockPilot. It pulls the published image — no build step required.

  1. Check prerequisites

    Docker Engine with the Compose v2 plugin (docker compose). On Windows/macOS, Docker Desktop covers this.

  2. Get the Compose file

    Clone the repo, or create a compose.yaml yourself:

    git clone https://github.com/stoxello/dockpilot.git
cd dockpilot
    services:
  dockpilot:
    image: ghcr.io/stoxello/dockpilot:latest
    pull_policy: always
    container_name: dockpilot
    restart: unless-stopped
    ports:
      - "${DOCKPILOT_PORT:-8080}:8080"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - ./data:/app/data
      - ./keys:/app/keys
      - ./stacks:/opt/stacks

  3. Start it

    docker compose up -d

    To use a different host port, set DOCKPILOT_PORT (for example in a .env file next to compose.yaml): DOCKPILOT_PORT=9000.

  4. Sign in

    Open http://localhost:8080, create the initial administrator account, and sign in. Public registration is disabled after that — only this first-run account can be created without authentication.

Private GHCR package?

Run docker login ghcr.io once before docker compose up -d, or add the credential through DockPilot's registry credentials page after first sign-in.

Multi-host

Remote agents

Install DockPilot.Agent on another Docker host when you want one DockPilot Control Center to manage multiple engines. The agent is a small, token-protected HTTP API that runs Docker CLI commands against the host it's installed on — it never talks back to the Control Center on its own.

Deploy the agent

  1. On the remote Docker host

    curl -O https://raw.githubusercontent.com/stoxello/dockpilot/main/compose.agent.yaml
printf "DOCKPILOT_AGENT_TOKEN=$(openssl rand -hex 32)\n" > .env
docker compose -f compose.agent.yaml up -d

  2. Register it in DockPilot

    Open DockPilot → Environments → Connect a Docker host, and add:

    • Name — a friendly label for the host
    • Endpoint — http://<remote-host>:5151
    • Access token — the DOCKPILOT_AGENT_TOKEN value from the remote host's .env

  3. Switch environments

    Use the top-bar environment selector to switch between This host and registered agents. Containers, Stacks, Images, Networks, Volumes, and Events follow the selected environment; stack creation and app catalog installs deploy to it too.

Stacks created for an agent

Stored under /opt/dockpilot/stacks on that host by default. Set DOCKPILOT_AGENT_STACKS_ROOT in the agent's .env to use a different absolute host path.

What the agent exposes

A Bearer-token-authenticated HTTP API the Control Center calls on your behalf:

  • GET /health
  • POST /api/stacks/deploy
  • POST /api/stacks/files / /save
  • POST /api/docker/commands
  • POST /api/registry/login / /logout
  • POST /api/replication/files
  • POST /api/replication/volumes/import / /export
⚠
Troubleshooting a restarting agent Check that .env sits next to compose.agent.yaml and contains a non-empty DOCKPILOT_AGENT_TOKEN — a missing token makes the agent exit on startup. 
docker compose -f compose.agent.yaml config
docker logs --tail 100 dockpilot-agent

If logs show exec /usr/bin/dotnet: input/output error, verify host Docker storage: docker run --rm mcr.microsoft.com/dotnet/aspnet:9.0 dotnet --info. If that also fails, run scripts/repair-docker-dotnet-io.sh on the Ubuntu host — normally first, and only with --reset-docker-storage if Docker still can't execute the base image.

Stack resilience

Backup & replication

Both features run from a scheduled plan you create once; DockPilot then runs it in the background.

Backup — local snapshots

Go to Backup, pick a source stack, and choose a schedule (every 15 min up to daily). Set how often a full backup runs versus an incremental one, and how many full backup chains to retain — older chains are pruned automatically after a new full backup completes.

Volumes Record images Stop stack for snapshot

Replication — cold standby

Go to Replication, pick a source stack and a registered target agent, and choose a schedule. Volumes are streamed to the target as a tar archive over the agent API; a bandwidth cap can be stored with the job.

Images Volumes Stop source for snapshot Bandwidth cap

Account policy: passwords must be at least 10 characters. Email confirmation is disabled (no SMTP configured), so accounts are usable immediately after creation.

Persistent data

The Compose file mounts the Docker socket plus three writable directories. Keep ./data, ./keys, and ./stacks to preserve state across upgrades.

Host pathContainer pathPurpose
/var/run/docker.sock/var/run/docker.sockLets DockPilot drive the host Docker engine
./data/app/dataSQLite database, registered agents, replication plans, and local replica metadata
./keys/app/keysData-protection keys — persisting these keeps logins valid across restarts
./stacks/opt/stacksCompose stacks created and managed by DockPilot
⚿
Docker socket = host admin DockPilot mounts /var/run/docker.sock. Access to this socket is equivalent to administrative control of the Docker host. Keep DockPilot behind a reverse proxy that terminates HTTPS, never expose it directly to the public internet, and treat anyone who can reach the UI as a host administrator. Public registration stays disabled — only the first-run administrator can be created without authentication.
Stay current

Updating & deploying

Update the Control Center

DockPilot's Updates page compares its running image digest against the published one and badges the sidebar when a newer version exists. From the directory containing your compose.yaml:

docker compose pull
docker compose up -d

DockPilot restarts on the new image. Your data, keys, and stacks are preserved by the mounted volumes. One-click self-update is on the roadmap — today you run the two commands above yourself.

Update an agent

Same idea, from the directory containing compose.agent.yaml on the remote host:

docker compose -f compose.agent.yaml pull
docker compose -f compose.agent.yaml up -d

The image uses pull_policy: always, so a plain up -d after a fresh pull is enough to move to the latest published agent build.

Build the Control Center from source

To build the image locally instead of pulling from GHCR, layer the build overlay over the base Compose file:

docker compose -f compose.yaml -f compose.build.yaml up -d --build

This builds from the repository Dockerfile — a multi-stage build on the .NET 9 SDK that also copies the Docker CLI and Compose plugin into the runtime image.

Local development

Requires the .NET 9 SDK and a running Docker Desktop or Docker Engine.

dotnet run --project ./src/DockPilot.Web

Open http://localhost:5146 (or https://localhost:7142), create the initial account, and sign in. Docker must be running for engine data and deployments to work.

Run the test suite

dotnet test

The tests/DockPilot.Web.Tests project covers the app catalog, app deployment, stack, setup, resource (image/network/volume), agent registry, backup, replication, registry credential, audit, and update services.

Publish the image (maintainers)

The public image is published to GitHub Container Registry as ghcr.io/stoxello/dockpilot, using a GitHub Personal Access Token with the write:packages scope, supplied through an environment variable — never hard-coded or committed.

$env:GHCR_TOKEN = "<github-pat-with-write:packages>"
.\ghcr-publish.ps1 -Owner stoxello -Version 0.1.0

The script logs in to ghcr.io, builds from the local Dockerfile, pushes the version tag, and also updates :latest (pass -NoLatestTag to skip that). It logs out afterward so credentials don't linger in your Docker config. The same script publishes the remote agent image:

.\ghcr-publish.ps1 -Owner stoxello -Image dockpilot-agent -Version 0.1.0
